# gadyuka_openrouter

[English](README.md) · **Русский**

**MCP‑сервер**, который позволяет ИИ‑агентам создавать изображения через
[OpenRouter Image API](https://openrouter.ai/docs/guides/overview/multimodal/image-generation) —
на **любой** image‑модели OpenRouter — со встроенным **учётом расходов**, **кэшированием**,
**батчами/цепочками** и поддержкой **референс‑изображений (image‑to‑image)**.

Указываешь один ключ OpenRouter — и агент генерит визуалы по запросу: посты для соцсетей,
hero‑баннеры для сайтов, обложки для блогов, продуктовые и маркетинговые картинки, переключает
модели, держится в рамках бюджета и бесплатно переиспользует прошлые результаты.

> Статус: **основа (v0.1)** — локальный `stdio`‑сервер, один ключ, **только генерация изображений**.
> HTTP‑транспорт с мульти‑тенантностью и выдачей ключей — в планах (см. ниже).

---

## Возможности

- **Любая image‑модель** — каталог тянется **вживую** из OpenRouter и кэшируется, без хардкода
  (GPT Image 2, Nano Banana 2 / Pro, FLUX.2, Seedream 4.5, Recraft, Riverflow, GPT‑5.x Image, …).
- **Сохранение на диск + manifest** — картинки пишутся в папку; агенту возвращаются пути и метаданные
  (модель, цена, seed), а не тяжёлый base64 в чат.
- **Учёт расходов и бюджет** — реальная стоимость каждого вызова пишется в локальный леджер; можно
  задать дневной/месячный/общий лимит с предупреждением или блокировкой.
- **Кэширование** — идентичный запрос (модель + промпт + параметры + байты референса) возвращает
  сохранённые файлы бесплатно, без повторной оплаты.
- **Батчи / цепочки** — много промптов или много вариаций одного промпта с ограничением параллелизма;
  частичные сбои сообщаются по каждому элементу.
- **Референс‑изображения** — image‑to‑image / перенос стиля из локального пути, URL или data‑URL.
- **Долгие задачи** — генерация занимает от секунд до ~3 минут; клиент ждёт с запасом по таймауту.

## Установка

```bash
npm install

```

## Подключение MCP‑клиента

Добавь сервер в своего агента. Пример (универсальный MCP `stdio`‑конфиг — работает с Cursor, VS Code и любым MCP‑клиентом):

```json
{
  "mcpServers": {
    "gadyuka_openrouter": {
      "command": "node",
      "args": ["/absolute/path/to/gadyuka_openrouter/dist/index.js"],
      "env": {
        "OPENROUTER_API_KEY": "sk-or-..."
      }
    }
  }
}
```

Единственное обязательное — твой ключ OpenRouter. У всего остального есть разумные значения по умолчанию.

### Переменные окружения

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `OPENROUTER_API_KEY` | — | **Обязательна для генерации.** Список моделей работает и без неё. |
| `GADYUKA_MODEL` | `openai/gpt-image-2` | Модель по умолчанию, если в вызове не указана `model`. |
| `GADYUKA_DATA_DIR` | `~/.gadyuka_openrouter` | Корень для кэша, леджера, бюджета. |
| `GADYUKA_OUTPUT_DIR` | `<data dir>/outputs` | Папка, куда по умолчанию пишутся картинки. |
| `GADYUKA_TIMEOUT_MS` | `300000` | Таймаут запроса (медленные модели — минуты). |
| `GADYUKA_MAX_RETRIES` | `2` | Повторы на 429/5xx с бэкоффом. |
| `OPENROUTER_BASE_URL` | `https://openrouter.ai/api/v1` | База API. |

## Инструменты

| Инструмент | Что делает |
|---|---|
| `list_image_models` | Список image‑моделей (вживую + кэш), фильтр по подстроке. |
| `get_image_model` | Точные параметры и цены по провайдерам для одной модели. |
| `generate_image` | Генерация 1–N картинок из промпта (+ опц. референсы). Сохраняет на диск. |
| `generate_batch` | Много промптов или N вариаций одного промпта, параллельно. |
| `get_usage` | Итоги расходов, разбивка по моделям, экономия от кэша, остаток бюджета. |
| `set_budget` | Задать/снять лимит трат (`warn` или `block`). |

Гайд для ИИ‑агента: [`skills/gadyuka-openrouter/SKILL.md`](skills/gadyuka-openrouter/SKILL.md).

### Быстрые примеры (аргументы инструментов)

Одна качественная картинка:
```json
{ "name": "generate_image", "arguments": { "prompt": "neon city at dusk, cinematic", "aspect_ratio": "16:9" } }
```

10 вариантов за один вызов (модель должна поддерживать `n>1`, напр. GPT Image 2):
```json
{ "name": "generate_image", "arguments": { "prompt": "minimalist logo, single line", "n": 10 } }
```

«Один стиль на несколько постов» через батч (разные промпты, общий референс):
```json
{ "name": "generate_batch", "arguments": {
  "prompts": ["instagram square hero", "story 9:16", "website banner 16:9"],
  "reference_images": ["/path/to/style-ref.jpg"],
  "model": "google/gemini-3.1-flash-image-preview"
} }
```

Оценить стоимость до траты:
```json
{ "name": "generate_image", "arguments": { "prompt": "...", "model": "bytedance-seed/seedream-4.5", "n": 4, "dry_run": true } }
```

## Как работает кэш

Ключ кэша — хэш от `модель + промпт + влияющие на пиксели параметры + байты референса`
(+ «соль варианта» для сэмплов батча с идентичным запросом). При попадании файлы копируются в твою
папку вывода и записывается `$0.00`. `force: true` обходит кэш (и блокировку бюджета) и всегда
генерит заново. `seed` делает запрос воспроизводимым.

## О расходах

Стоимость берётся из `usage.cost` каждого ответа и пишется в JSONL‑леджер. Предварительная оценка
(`dry_run`, проверка бюджета) использует пер‑image цену, где она опубликована; **модели с биллингом по
токенам/мегапикселям (напр. GPT Image 2, Gemini Image) заранее оценить нельзя** — их реальная цена всё
равно фиксируется после вызова. Модели с ценой за картинку (Seedream, FLUX, …) оцениваются чётко.
